# WeightedInstFreq · Frontend Dashboard

A React + Vite visualization dashboard for the **Weighted Instruction Frequency LLVM Pass** (`-weighted-inst-freq`).

---

## What This Frontend Does

This dashboard reads `results.csv` — generated by the LLVM pass — and renders:

| Feature | Description |
|---|---|
| **CSV Upload** | Drag-and-drop or click-to-upload `results.csv` parsed in-browser via PapaParse |
| **Sample Data** | One-click demo with embedded sample CSV (no file needed) |
| **KPI Cards** | Total functions, total weighted cost, hottest function, highest cost category, BB count |
| **Stacked Bar Chart** | Per-function weighted cost broken down by instruction category |
| **Category Pie Chart** | Overall cost distribution across all instruction types |
| **Hotspot Table** | Functions ranked by cost with configurable hotspot threshold |
| **Basic Block Table** | Per-BB cost totals sorted descending |
| **Filters** | Filter by function, category, and adjust hotspot threshold |
| **Explanation Panel** | Project overview and LLVM pipeline diagram |

---

## How It Connects to the LLVM Project

```
test.c
  └─▶ clang-14 -O0 -S -emit-llvm test.c -o test.ll
         └─▶ opt-14 -enable-new-pm=0 \
                  -load ./build/libWeightedInstFreq.so \
                  -weighted-inst-freq \
                  -weight-file=./weights.cfg \
                  -hot-threshold=30 \
                  -out-file=results.csv \
                  -disable-output test.ll
                    └─▶ results.csv
                              └─▶ [ THIS DASHBOARD ]
```

The frontend is purely a **visualization layer**. It does not run LLVM commands. The browser reads the CSV produced by the pass and renders charts and tables.

1. Run the LLVM pass first (see above), which generates `results.csv`
2. Open the dashboard in the browser
3. Click **"Upload CSV"** or drag-and-drop `results.csv` onto the upload zone
4. Or click **"Load Sample Data"** for a demo with embedded sample data

## What Each Chart Means

### Stacked Bar Chart — Function Cost Breakdown
- X-axis: function names from the IR
- Y-axis: total weighted cost = Σ(count × weight)
- Each color represents one instruction category (Arithmetic, Memory, etc.)
- Only uses rows where `basic_block = FUNCTION_TOTAL`

### Category Pie Chart — Category Distribution
- Shows what fraction of total cost comes from each instruction type
- Useful for identifying whether the program is memory-bound, compute-bound, etc.

### Hotspot Table
- Lists all functions with their total cost, dominant category, and hotspot status
- A function is a **HOTSPOT** if `totalCost >= threshold`
- Adjust the threshold using the filter input (default: 30)

### Basic Block Table
- Lists individual basic blocks (rows where `basic_block ≠ FUNCTION_TOTAL`)
- Groups by (function, basic_block) and sums cost
- Sorted descending by total cost
- Color bar shows relative cost

## Troubleshooting

| Issue | Fix |
|---|---|
| `npm: command not found` | Install Node.js from https://nodejs.org |
| Port 5173 in use | Run `npm run dev -- --port 3000` |
| CSV not loading | Make sure the file has the header: `function,basic_block,group,count,weight,cost,pct` |
| Charts empty after upload | Check that the CSV has rows where `basic_block = FUNCTION_TOTAL` |
| White/broken layout | Run `npm install` first |

## Viva Explanation

> "The frontend is only a visualization layer. The actual core project is the LLVM command-line analysis pass written in C++ (`WeightedInstFreq.cpp`). The pass runs on LLVM IR, classifies instructions, applies configurable weights, and writes `results.csv`. The browser cannot execute LLVM commands — it only reads the CSV and renders charts. The frontend was built using React, Vite, Recharts for charts, and PapaParse for in-browser CSV parsing. No backend server, no database, no Docker — purely static."

## CSV Format Reference

```
function,basic_block,group,count,weight,cost,pct
```

| Column | Meaning |
|---|---|
| `function` | IR function name |
| `basic_block` | `FUNCTION_TOTAL` (summary) or BB label (e.g. `BB0`, `BB1`) |
| `group` | Instruction category: Arithmetic / Memory / ControlFlow / Call / Cast / Comparison / Other |
| `count` | Number of instructions of this type |
| `weight` | Configured weight for this category |
| `cost` | `count × weight` |
| `pct` | Percentage of this group within the function |
